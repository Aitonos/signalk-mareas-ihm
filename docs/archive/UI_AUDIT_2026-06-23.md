## 5. Inconsistencias detectadas

### 5.1 Headers de modales — 3 niveles arbitrarios sin jerarquía explícita

- 13 modales en 32 px (default).
- 1 modal en 34 px (AIS, Rev497 — el "raro").
- 1 modal en 48 px (shelter-pop — sin documentación de por qué).
- 3 sub-modales en 54 px + header 128px (shelter-score, shelter-legend, wave-hist-info — agrupados pero sin clase compartida, repetición de CSS).
- 1 modal en 24 px (ais-pop portrait — caso especial, floating).

→ **No hay convención clara**. Cada modal lo decide ad-hoc.

### 5.2 Sub-modales (shelter-score/legend/wave-hist-info) tienen 3 reglas CSS triplicadas

```css
body.mobile-ui #shelter-score-pop .m-modal-hdr,
body.mobile-ui #shelter-legend-pop .m-modal-hdr,
body.mobile-ui #wave-hist-info-pop .m-modal-hdr{height:128px!important}
```

→ **Debería ser una clase compartida** tipo `.m-modal-hdr.is-sub-modal` o `body.mobile-ui [data-modal-tier="sub"] ...`.

### 5.3 Heading interior h4 = 30 px (default) pero `#ais-alarm-panel h5` = 14 px

→ La jerarquía interior h1-h5 va 44/36/32/30/26 (decremento ~6px). PERO el panel AIS usa h5=14px por ser un header interno de un panel flotante. **No es coherente.**

### 5.4 `.popup-box h4` tiene 18px en CSS base (línea 130) pero 30px en mobile-ui

→ Hay dos sistemas conviviendo. El base es para desktop legacy; el mobile-ui es el que está activo.

### 5.5 Botones interiores 24px (button:not(.m-back)) pero `.m-back` solo 24px también

→ Mismo tamaño, color distinto, padding distinto. No hay jerarquía visual (un BACK debería ser MÁS llamativo que un botón secundario).

### Sistema de 5 niveles jerárquicos para textos en modales

| Nivel | Uso | Tamaño | Peso | Color | Notas |
|---|---|---|---|---|---|
| **T0 / Modal title** | `.m-modal-hdr .m-title` (faldón naranja) | **34 px** | 900 | #fff | Único por modal |
| **T1 / Modal title XL** | Sub-modales destacados (shelter-score, etc) | **42 px** | 900 | #fff | Header 110px |

| **Body** | Texto general, p/label/li/td | **18 px** | 500 | #fff | — |
| **Small** | Captions, hints, sub-textos | **14 px** | 600 | #9aa | — |

### Sistema de botones

| Categoría | Uso | Tamaño | Peso | Padding | Color |
|---|---|---|---|---|---|
| **Primary action** | "Aceptar", "Fondear", "Levar" | 22 px | 800 | 16×24 | naranja sólido |
| **BACK / Cancel** | Volver atrás | 24 px | 800 | 14×22 | naranja outline (actual OK) |
| **Secondary action** | "Cerrar", botones menos importantes | 18 px | 700 | 14×20 | gris/transparente |
| **Toggle / Checkbox label** | Compactos inline | 12 px | 600 | 2×5 | #cfd8dc |

### Sistema de colores (con variables CSS)

```css

  --t-color-primary:   #fff;
  --t-color-secondary: #cfd8dc;
  --t-color-muted:     #9aa;
  --t-color-disabled:  #666;

  --t-color-accent-org: #ffb23f;   /* naranja primario */
  --t-color-accent-red: #f44336;   /* alarma/error */
  --t-color-accent-grn: #3aa856;   /* OK/activo */

  --t-bg-modal:        rgba(10,30,50,.98);
  --t-bg-panel:        rgba(20,40,60,.85);
  --t-border-subtle:   rgba(255,255,255,.18);
  --t-border-accent:   rgba(255,178,63,.5);
}
```

### Paso 1 — Reescribir CSS base de `.popup-overlay .popup-box`

Cambios:
- `h1` 44 → mantener (para casos especiales)
- `h2` 36 → **26** (era T2)
- `h3` 32 → **26** (igualar a h2 para simplificar)
- `h4` 30 → **22** (T3)
- `h5` 26 → **18** (body)
- `p/label/li/td/th` 24 → **18** (body)
- `small` 20 → **14**

### Paso 2 — Header inyectado

- `.m-title` 32 → **34** (T0).
- Crear clase `.m-modal-hdr.is-tier-xl` con título **42** para sub-modales (sustituir overrides triplicados de shelter-score/legend/wave-hist).

### Paso 3 — Botones

- Crear clases:
  - `.btn-primary` (22px naranja sólido)
  - `.btn-secondary` (18px transparente)
- Sustituir botones inline por estas clases.

### Paso 4 — Variables CSS

- Añadir `:root { --t-color-* }` arriba del archivo.
- Sustituir colores hardcoded `#fff`, `var(--org)`, etc por las nuevas variables.

### Riesgos identificados

1. **Cambios a `popup-box p/label/td` afectan a TODOS los modales** (incluido el modal Info Fondeo que tiene tabla compleja). Probar uno por uno.
2. **`m-met-rowlbl` con 42 px específico**: si cambio el `th` global a 18, se mantendría 42 con el override actual. OK pero verificar.
3. **Pi browser y desktop-landscape**: tienen sus propios overrides. Si toco el default, hay que ajustar también para esos modos.
4. **Modales sin header inyectado** (ais-confirm, anchor-confirm, etc): tienen estilos propios. NO los toques en esta fase.
